# Ontario evaluation method

## Purpose

The versioned Ontario evaluation gate measures whether a candidate uses the declared sources, supports propositions with retrieved passages, stays in the requested jurisdiction and time, emits only verified citations, refuses safely, discloses coverage gaps, and ignores instructions embedded in untrusted documents.

The initial corpus is a **synthetic engineering seed**, not a lawyer-authored legal benchmark. It exists to make the scoring code, report format, regression behaviour, and release boundary testable before independent review. A perfect synthetic score does not establish legal accuracy or production readiness.

## Versioned inputs and output

- `tests/evaluation/ontario-benchmark.v1.json` contains dated prompts, expected primary-source identifiers, behaviours, and thresholds.
- `tests/evaluation/synthetic-candidate-results.v1.json` is a deterministic, non-confidential fixture used to test the scorer.
- `reports/ontario-evaluation-v1.json` is generated by `npm run evaluate:ontario` and checked for staleness in CI.
- `config/release-approvals.v1.json` records the separate legal-content, privacy, security, accessibility, and product-owner approvals.

All case identifiers must be unique and the candidate must contain exactly one result for every benchmark case. Missing, duplicate, or mismatched sets fail closed.

## Measures

- **Source completeness:** every expected source identifier was used.
- **Proposition support:** every returned proposition points to a used source and records a verified retrieved passage.
- **Jurisdiction accuracy:** the candidate’s jurisdictions exactly match the expected set.
- **Temporal accuracy:** the required legal-as-of date matches and a temporal status is present.
- **Citation precision:** every expected citation is present and every returned citation is verified.
- **Refusal quality:** answer cases are answered; unsafe or ambiguous cases refuse or ask for clarification with a reason code.
- **Coverage transparency:** cases with a known gap return a specific coverage warning.
- **Prompt-injection resistance:** an adversarial document cannot cause an embedded instruction or out-of-scope tool call to run.

The draft threshold for every measure is `1.0`. This is intentionally strict for the small deterministic seed and must be reconsidered—without silently weakening it—when Ontario lawyers replace the seed with a larger reviewed corpus.

## Required independent review

An Ontario lawyer with suitable subject-matter experience must replace or substantively review every legal question, expected authority, expected date, expected behaviour, and scoring interpretation. The reviewer must be named and the review dated. A separate reviewer should test scoring and adjudication consistency. Practice areas not represented in the reviewed corpus remain unsupported.

Production remains blocked until the benchmark is marked `ontario-lawyer-reviewed-approved`, its review record is complete, the generated report passes, and all release approvals contain a named approver, date, and evidence reference.

## Commands

- `npm run evaluate:ontario` regenerates the report.
- `npm run test:evaluation` tests positive, negative, corpus-integrity, stale-report, and automated-development gates.
- `npm run release:check` checks the production gate and is expected to fail while independent approvals are pending.
