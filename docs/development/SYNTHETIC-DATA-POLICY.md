# Synthetic Data Policy

> NOT APPROVED FOR REAL OR CONFIDENTIAL DATA

## Policy

Until the Security Gate (Sprint 14) is passed with a GO decision:

1. Only synthetic documents may be used.
2. No real client data, contracts, NDAs, or privileged content.
3. No real user accounts from Atlas or clients.
4. No production API keys for LLM providers.
5. No real Supabase project — only local Supabase.

## Synthetic Fixtures

Location: `test/fixtures/documents/`

All fixtures contain the watermark:
```
SYNTHETIC TEST DOCUMENT — NO REAL CLIENT DATA
```

Fixtures are generated by `test/fixtures/documents/generate-fixtures.py` using
ReportLab, python-docx, and openpyxl. No real documents are copied.

## Permitted Data

- Synthetic users created via Supabase local Auth
- Synthetic projects and documents
- Disposable LLM API keys (local only, not committed)
- MinIO local credentials (minioadmin/minioadmin)

## Prohibited Data

- Real Atlas client documents
- Real employee credentials
- Production Supabase credentials
- Production LLM API keys
- Any PII (personal identifiable information)
- Any legally privileged content
