# Local Development

> NOT APPROVED FOR REAL OR CONFIDENTIAL DATA

## Prerequisites

- Node.js 20.x (see `.nvmrc`)
- npm 10.x
- Docker Desktop (or Docker Engine + Docker Compose)
- Git
- Python 3 (for fixture generation only)

## Quick Start

```bash
# 1. Check prerequisites
make doctor

# 2. Bootstrap (install deps, start Supabase, MinIO, generate secrets)
make bootstrap

# 3. Start the application
make dev

# 4. Verify
make smoke-local
```

## Services and URLs

| Service    | URL                              | Notes                        |
|------------|----------------------------------|------------------------------|
| Frontend   | http://localhost:3000            | Next.js dev server           |
| Backend    | http://localhost:3001            | Express API                  |
| MinIO API  | http://localhost:9000            | S3-compatible storage        |
| MinIO Console | http://localhost:9090         | MinIO web UI                 |
| Supabase   | http://localhost:54321           | Auth, Postgres, API          |

All services bind to 127.0.0.1 only.

## Environment Variables

See `docs/development/ENVIRONMENT-VARIABLES.md` for the full catalog.

Local `.env` files are generated by `make bootstrap`:
- `backend/.env` — backend secrets and config
- `frontend/.env.local` — frontend public config

Never commit these files.

## Optional: LLM Provider Key

The application starts without any LLM key. Chat features will return a
functional error message. To test chat locally:

1. Add a disposable key to `backend/.env`:
   ```
   ANTHROPIC_API_KEY=sk-ant-...
   ```
2. Restart: `make stop && make dev`
3. Run: `make smoke-local-ai`

## Logs

```bash
make logs          # tail all services
docker-compose logs -f backend    # specific service
```

## Stop and Reset

```bash
make stop          # stop services, keep data
make reset CONFIRM=local-only   # destroy local data only
```

## Database Commands

```bash
make db-status     # show migration status
make db-verify     # verify schema objects
make db-reset CONFIRM=local-only   # reset local database
```

## Updating Dependencies

```bash
cd backend && npm ci   # deterministic install from lockfile
cd frontend && npm ci
```

Never use `npm install` in CI or bootstrap. Always use `npm ci`.

## Architecture

```
Browser → Frontend (Next.js :3000) → Backend (Express :3001)
                                        ├── Supabase (Auth + Postgres :54321)
                                        ├── MinIO (S3 storage :9000)
                                        └── LibreOffice (in backend container)
```
